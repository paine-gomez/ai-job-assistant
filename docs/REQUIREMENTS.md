# 需求规格文档

## 项目背景

大四应届生曹嘉明（AI产品经理方向）需要一个 AI 求职助手作品集项目，
用于展示 AI 产品能力，作为简历加分项和面试 Demo。

---

## 功能模块一：知识库问答 📚

### 用户故事
> 作为求职者，我想上传求职相关资料（简历、行业报告等），然后向 AI 提问，
> AI 能基于我上传的资料给出准确回答，方便面试前快速回顾。

### 功能描述
1. 用户可上传 PDF、Word（.docx）、TXT 格式文件
2. 上传后自动解析文件内容，提取纯文本
3. 用户在对话界面输入问题
4. AI 基于已上传文档的内容回答，并注明来自哪份文档
5. 如果答案不在文档中，AI 应明确表示"资料中未提及"（MVP 用 Prompt 控制，不做精确溯源标注）
6. 对话支持流式输出（逐字显示，体验更好）
7. 用户可查看、删除已上传的文档

### 验收标准
- [ ] 可上传 PDF 文件并成功解析
- [ ] 可上传 Word 文件并成功解析
- [ ] 提问后 AI 回答基于文档内容
- [ ] 遇到文档外的问题，AI 明确说不知道
- [ ] 回答是流式逐字显示的
- [ ] 可查看文档列表
- [ ] 可删除不需要的文档
- [ ] 上传非支持格式时有提示

## 功能模块二：JD 分析工具 ⚡

### 用户故事
> 作为求职者，我看到一份招聘 JD，想快速了解这个岗位的核心要求、
> 薪资范围、所需技能等信息，而不是逐字阅读长文。

### 功能描述
1. 用户粘贴 JD 文本到输入框（支持长文本）
2. 点击分析，AI 提取结构化信息
3. 返回格式化的分析结果，包含：
   - 公司名称/行业
   - 岗位名称
   - 薪资范围（如有）
   - 学历要求
   - 经验要求
   - 核心技能要求
   - 岗位职责摘要
   - 关键词标签
   - 整体岗位总结

### 验收标准
- [ ] 粘贴 JD 后可成功分析
- [ ] 返回结果包含上述所有字段
- [ ] 分析结果格式清晰，易于阅读
- [ ] 空输入时有提示
- [ ] 非 JD 内容（如随便粘贴一段文字）有合理处理

## 功能模块三：简历匹配工作流 🔗

### 用户故事
> 作为求职者，我想知道我的简历和某个岗位的匹配程度，
> 以及具体哪里不够好，应该怎么改进。

### 功能描述
1. 用户上传简历文件（PDF/Word），或直接粘贴简历文本
2. 用户粘贴目标 JD 文本
3. 点击匹配，AI 从四个维度评分：
   - 技能匹配度
   - 经验匹配度
   - 学历匹配度
   - 综合素质匹配度
4. 返回总评分 + 各维度分数 + 评语
5. 列出优势项（匹配良好的部分）
6. 列出差距项（需要提升的部分）
7. 给出简历优化建议
8. 给出面试准备建议

### 验收标准
- [ ] 可上传简历文件进行分析
- [ ] 四个维度分别评分（百分制）
- [ ] 总评分合理（不是乱打的分）
- [ ] 优势项和差距项具体、有针对性
- [ ] 优化建议切实可用
- [ ] 面试建议有参考价值
- [ ] 缺简历或缺 JD 时有提示

## 非功能需求

| 类别 | 要求 |
|------|------|
| 界面 | 专业、简洁、现代，符合 AI 产品审美 |
| 响应 | AI 回答采用流式输出，首字延迟 < 3秒 |
| 错误处理 | 所有错误有中文友好提示，不显示技术报错 |
| 文件限制 | 单文件 ≤ 5MB，支持 PDF/DOCX/TXT |
| 运行环境 | 本地开发服务器，浏览器访问 localhost |
| API 容错 | DeepSeek API 不可用时，返回友好提示而非白屏 |
| 数据安全 | API Key 仅存服务端环境变量，绝不出现在前端代码中 |
